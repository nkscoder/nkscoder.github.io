Building AI Bank Statement Analysis with Ollama on NVIDIA GPU

By Nitesh Kumar Singh · Senior Software Developer, Quess Corp Ltd (Airtel Payments Bank) · June 10, 2026

At Airtel Payments Bank through Quess Corp Ltd, I lead development of an AI-powered bank statement analysis platform. The goal: ingest thousands of statements in PDF, CSV, and XML formats, extract transactions accurately, categorize spending, detect risk indicators, and resolve party entities — all at scale, with data privacy requirements that ruled out external LLM APIs.

The Challenge

Bank statements arrive in wildly inconsistent formats. Tables break across pages, OCR quality varies, and transaction descriptions are noisy. Traditional rule-based parsers fail on edge cases. Cloud LLM APIs introduced latency, cost, and compliance concerns for sensitive financial data.

We needed a system that could:

  • Process high volumes of statements concurrently
  • Run LLM inference on-premises with full data control
  • Integrate with existing Django REST APIs and PostgreSQL storage
  • Scale horizontally as upload volume grew

Solution: Ollama on NVIDIA GPU

We deployed Ollama for local LLM inference on NVIDIA GPU hardware. This gave us CUDA-accelerated batch processing with configurable VRAM allocation, model warm-loading, and tunable context windows for long statement extracts.

Key GPU configuration decisions:

  • CUDA drivers & VRAM tuning — right-sized models for available GPU memory
  • Batch inference — grouped prompts to maximize GPU utilization
  • Model selection — balanced accuracy vs. throughput for financial text
  • Fallback pipelines — Pandas-based extraction when LLM confidence is low

Backend Architecture

The platform is built on a proven Python stack:

  • Django REST Framework — upload APIs, job status, result retrieval
  • Celery + RabbitMQ — async task queues for ingestion and LLM jobs
  • Ray — distributed processing for large CSV/XML batches
  • PostgreSQL — normalized transaction storage with audit trails
  • Pandas — data transformation and validation layers

Upload flow: client submits statement → Celery worker parses raw file → Ray distributes chunks → Ollama GPU worker runs LLM categorization → results validated → PostgreSQL persist → REST API response.

AI Capabilities Delivered

  • Intelligent transaction categorization (income, EMI, transfers, merchant types)
  • Risk indicator detection (bounce patterns, circular transfers, salary consistency)
  • Party entity resolution across multiple statement formats
  • Structured JSON output for downstream credit and underwriting systems

Results & Lessons

Local GPU inference reduced per-statement processing cost compared to cloud APIs while keeping data on-premises. The hybrid approach — LLM for ambiguous cases, rules for structured data — delivered the best accuracy-to-speed ratio.

Three lessons for similar fintech AI projects:

  1. Start with GPU profiling before choosing model size
  2. Design idempotent Celery tasks — statement reprocessing is inevitable
  3. Log LLM confidence scores; human review queues save production incidents
